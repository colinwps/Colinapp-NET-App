# Colinapp 通用框架 · 架构规划

> 一个通用的后台管理框架基础设施。后端 ASP.NET Core (.NET 10)，前端 Vue 3 + Element Plus。
> 提供组织管理（人员、职位、部门、权限）与平台设置（日志、菜单、字典、参数）等基础能力，并支持功能与菜单的灵活扩展。

## 1. 技术选型

| 维度 | 选型 | 说明 |
|------|------|------|
| 后端框架 | ASP.NET Core (.NET 10) | Web API |
| ORM | EF Core 10 | Code First + 迁移 |
| 数据库 | MySQL | 驱动 Pomelo.EntityFrameworkCore.MySql |
| 架构风格 | 模块化分层 | Api / Application / Domain / Infrastructure + Shared |
| 多租户 | 预留 | 实体基类内置 `TenantId`，配合全局查询过滤器，初期可关闭 |
| 认证 | JWT | access token + refresh token |
| API 文档 | Swagger / Scalar | |
| 日志 | Serilog | 结构化日志，输出到文件 + 数据库 |
| 校验 / 映射 | FluentValidation / Mapster | |
| 前端 | Vue 3 + Element Plus | Vite + TypeScript + Pinia + Vue Router |
| 前端通信 | Axios | 拦截器统一处理 token / 错误码 |
| 缓存（可选） | Redis | 配置 / 字典 / 权限缓存 |

## 2. 后端解决方案结构（模块化分层）

```
Colinapp.sln
├─ src/
│  ├─ Colinapp.Api                 # 启动层：Controllers、中间件、DI 注册、JWT、Swagger
│  ├─ Colinapp.Application         # 应用层：业务服务、DTO、接口、校验、权限服务
│  ├─ Colinapp.Domain              # 领域层：实体、枚举、领域常量、聚合（含 TenantId 基类）
│  ├─ Colinapp.Infrastructure      # 基础设施：EF Core DbContext、仓储、迁移、缓存、日志落库
│  ├─ Colinapp.Shared              # 通用层：统一响应、分页、异常、扩展方法、工具类
│  └─ Colinapp.Modules.*           # 可选业务模块（按需扩展，如 Modules.Workflow）
└─ test/
   └─ Colinapp.Tests
```

**依赖方向**：`Api → Application → Domain ← Infrastructure`，`Shared` 被各层引用。
新功能优先作为独立模块（`Modules.*`）加入，不污染核心。

## 3. 核心数据模型（组织管理 + 权限）

采用 **RBAC + 数据权限范围** 模型。

| 实体 | 说明 |
|------|------|
| `User`（人员） | 账号、姓名、所属部门、关联多个职位/角色 |
| `Department`（部门） | 树形结构（ParentId + 层级路径），支持负责人 |
| `Position`（职位） | 职位/岗位，可与部门、角色关联 |
| `Role`（角色） | 权限载体，绑定菜单/按钮权限 + 数据范围 |
| `Menu`（菜单/权限） | 树形，类型=目录/菜单/按钮，按钮即细粒度权限点 |
| `UserRole` / `RoleMenu` / `UserPost` | 多对多关联表 |

### 权限分两级

1. **功能权限** —— 角色 → 菜单/按钮（控制能否访问接口与显示按钮）。
2. **数据权限** —— 角色的数据范围（全部 / 本部门 / 本部门及下级 / 仅本人 / 自定义部门），由查询过滤器自动注入。

## 4. 平台设置功能

- **菜单管理** —— 维护目录/菜单/按钮树，绑定前端路由、组件路径、权限标识、图标、排序。
- **日志** —— 登录日志、操作日志（AOP 拦截 Controller 自动记录请求/耗时/结果）、系统异常日志。
- **数据字典** —— 通用键值字典，前端下拉统一来源。
- **参数配置** —— 系统级 Key-Value 配置，支持缓存。
- **租户管理（预留）** —— 实体基类内置 `TenantId`，配合 EF Core 全局查询过滤器实现租户隔离，初期可关闭。

## 5. 灵活扩展机制（关键设计）

1. **菜单驱动**：新增功能 = 后端建模块 + 前端建页面组件 + 在菜单管理里挂一个菜单项（指定组件路径与权限标识），无需改路由代码。
2. **动态路由**：前端登录后拉取菜单树，递归生成 Vue Router 路由，组件用 `import.meta.glob` 按路径懒加载。
3. **模块化解耦**：`Modules.*` 项目各自注册服务、迁移，主程序自动扫描装配（约定优于配置）。
4. **统一约定**：所有实体继承审计基类（`Id / CreatedBy / CreatedTime / UpdatedBy / IsDeleted / TenantId`），软删除 + 审计自动填充。

## 6. 横切关注点

- 统一响应体 `{ code, message, data }`，全局异常中间件兜底。
- JWT + 刷新令牌；接口鉴权用 `[Authorize]` + 自定义权限策略（校验菜单权限标识）。
- 软删除、审计字段、租户过滤通过 EF Core `SaveChanges` 拦截器与全局查询过滤器统一处理。
- Redis（可选）做配置 / 字典 / 权限缓存。

## 7. 前端结构（规划）

```
web/
├─ src/
│  ├─ api/            # 接口封装（按模块）
│  ├─ assets/
│  ├─ components/     # 通用组件
│  ├─ directives/     # v-auth 等指令
│  ├─ layout/         # 主框架布局
│  ├─ router/         # 静态路由 + 动态路由生成
│  ├─ stores/         # Pinia（user / permission / app）
│  ├─ utils/          # request 封装、工具
│  └─ views/          # 页面（与后端菜单组件路径对应）
└─ vite.config.ts
```

- 登录后拉取用户信息 + 菜单树 → 生成动态路由 + 左侧菜单。
- 按钮级权限通过 `v-auth="'权限标识'"` 指令控制显隐。
- Axios 拦截器统一注入 token、处理 401 刷新与统一错误码提示。

## 8. 开发路线（分阶段）

1. **阶段一 · 地基**：解决方案骨架、EF Core + MySQL 连通、统一响应/异常/审计基类、JWT 登录、Swagger。
2. **阶段二 · 组织与权限**：部门/职位/人员/角色/菜单 CRUD + RBAC 鉴权 + 数据权限。
3. **阶段三 · 平台设置**：菜单管理、日志、字典、参数配置。
4. **阶段四 · 前端框架**：登录、动态菜单/路由、布局、按钮权限指令、组织管理与平台设置页面。
5. **阶段五 · 扩展样例 + 租户开关**：一个示例业务模块演示扩展流程，启用多租户过滤。
